# Richie — Assets Page (AssetDocumentationPage.xaml)

## Page Overview
The **Assets** page is the central hub for managing, viewing, and analyzing all financial assets in the user’s portfolio. It presents:
- A browsable asset list (DataGrid)
- Real-time portfolio KPI cards (totals, profit/loss, diversification, top performer)
- Portfolio allocation insights (donut chart + allocation summary)
- Documentation coverage tracking (documented vs undocumented assets + actions)

A key capability of this page is that users can **include/exclude individual assets from portfolio calculations** using the **Excl.** checkbox. When an asset is toggled, **all KPI cards and charts refresh immediately** to reflect only non-excluded assets.

## UI Components & Features

### 1) Header Bar
Located at the top of the page, the header bar contains the page title **“Assets”** and three primary actions:
- **Goals** — opens the financial goals view.
- **Bulk Upload** — allows importing multiple assets at once.
- **Add Asset** — adds a new asset manually.

The **Add Asset** button is styled as the primary call-to-action to guide user flow.

### 2) KPI Cards Row
Directly under the header, the page displays a row of summary cards providing an at-a-glance view of portfolio health.

#### Total Portfolio Value Card
Shows:
- **Total current portfolio value**
- **Invested vs Current** breakdown
- **Profit/Loss** amount and **profit/loss percentage** badge

Color coding is accessibility-friendly:
- **Teal (#0F766E)** indicates profit / positive values
- **Orange (#EA580C)** indicates loss / negative values

#### Diversification Score Card
Displays portfolio concentration metrics using an HHI-style concentration score:
- Circular **score indicator** (0–100 range)
- **Classification label**: **Poor / Moderate / Good / Excellent**
- A **color-coded progress bar** representing score intensity

#### Top Performing Asset Card
Highlights the asset with the **highest rate of return**:
- Asset name
- Return percentage

#### Portfolio Insights Card
Presents a concise recommendation or observation generated by the application (rule-based or AI-driven), displayed alongside a **lightbulb** icon.

### 3) Charts & Documentation Section (Two-Column Layout)
A two-column layout provides both analytical visuals and documentation management.

#### Left: Asset Allocation (Donut Chart)
- **Donut chart** visualizing portfolio distribution by **asset type**.
- An interactive, **scrollable legend** lists each category with:
  - Color swatch
  - Category name
  - Value and percentage

Below the chart:
- **Asset Allocation Summary**
  - Largest Asset Category
  - Portfolio Concentration
  - Number of Asset Categories
  - Most Diversified Category

#### Right: Documentation Checklist
Tracks how many assets have supporting documents uploaded.

Includes:
- **Document coverage progress bar** (documented assets vs total)
- **Scrollable list of undocumented assets** showing:
  - Name
  - Type
  - **Add file** action button
- A **green success banner** when all assets are documented

### 4) Asset Data Table (DataGrid)
A full-width **DataGrid** lists all assets with the following columns:
- **Name**
- **Type**
- **Identifier**
- **Invested**
- **Current**
- **P&L**
- **P&L %**
- **Excl.** (exclusion checkbox)
- **Actions** (icon-only)

The **Actions** column contains compact icon-only buttons for:
- View
- SIP Scheduler
- Documents
- Edit
- Delete

### 5) Exclusion Checkbox (Excl.) — Real-Time Portfolio Updates
Each asset row includes an **Excl.** checkbox that allows the user to exclude/include that asset from portfolio calculations.

When toggled:
- The application updates the asset’s `IsExcludedFromPortfolio` state
- **All KPI cards and charts refresh instantly**
- The feature applies to **ALL asset types** (no restrictions by asset category)

This ensures the portfolio analysis always reflects the user’s intended set of included assets.

## Design Changes from Previous Version

### Color Palette Change
To improve accessibility and reduce cultural color bias:
- All **red** and **green** UI indicators have been removed.
- Positive/profitable values use **teal (#0F766E)**.
- Negative/loss values use **orange (#EA580C)**.

### Gap Reduction
Spacing between cards, sections, and rows was reduced to create a more compact, information-dense layout.

### Buttons Replaced with Icons
DataGrid action buttons were converted from text-based buttons to compact icon-only `SymbolIcon` buttons. This improves:
- Visual clarity
- Consistent sizing
- Better utilization of horizontal space

### Overall Page Layout Overhaul
The Assets page was reorganized into a consistent **4-row grid layout**:
1. Header
2. KPI Cards
3. Charts & Documentation
4. Asset DataGrid

The result is a modern card-based design with improved typography hierarchy and a consistent blue brand accent.

## Tech Stack
- **WPF** (.NET)
- **WPF UI** (Lepo.co)
- **LiveCharts (LiveCharts2)** with **SkiaSharp**
- **CommunityToolkit MVVM**
- **XAML** with custom converters and styles

## Notes for Developers
- The **Excl. checkbox** is bound to the asset’s exclusion state and triggers a portfolio KPI refresh.
- Portfolio KPIs and charts are computed from the portfolio summary returned by the Assets service.
- The DataGrid uses template columns to support the checkbox + icon-only actions.
